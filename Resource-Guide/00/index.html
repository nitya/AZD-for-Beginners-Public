    üëãüèΩ Welcome¶

    DECONSTRUCT TO LEARN!

    Level: Beginner-Intermediate
    Duration: 3-4 hours

    Building enterprise-grade AI solutions requires knowledge of design architecture, infrastructure provisioning, environment configuration, and more. As a beginner, where do you even start? In this workshop, we'll teach you how to use the Azure Developer CLI (azd) with AI App Templates to achieve this in 4 steps:

    1. Discover It - learn azd fundamentals, find the right azd "starter" template.
    2. Deploy It - use azd to deploy solution to Azure, validate that it works!
    3. Deconstruct It - use azd (and AI assistance) to understand codebase & workflow.
    4. Customize It - modify app source and infra resources to adapt it for your needs.

    Once you've complete this, challenge yourself to replicate the process with another AI App Template - and continue to build your intuition by using this repository as a learning sandbox.

    1. Before We Begin¶

    For the best experience, we recommend you start with three steps:

    1. Review pre-requisites - and create required accounts if needed.
    2. Launch GitHub Codespaces - get a pre-built development environment.
    3. Launch Mkdocs Preview - get a workshop guide in the browser.

    Then continue following workshop instructions in the browser-based guide. You can get the benefits of search, dark theme, and helpful admonitions that highlight key tasks, provide AI-assistance tips, and easy copy-paste functionality for code and commands.

    1.1 Review Pre-Requisites¶

    To get the most out of this workshop, you will needt the following:

    • Familiarity with - VS Code, Azure, Python & CLI tools
    • Azure Subscription - Create a free Azure account
    • Personal GitHub Account - Create one for free
    • GitHub Copilot Access - Get Started For Free
    • GitHub Codespaces - Free Tier is sufficient

    Congratulations! You are ready to begin the workshop! ‚úÖ

    1.2 Launch Codespaces¶

    You need a development environment with the right tools and dependencies installed. The repository is instrumented with a devcontainer.json that defines these requirements. You can activate this in GitHub Codespaces in just a few clicks:

    1. Start by creating a fork of this repo in your personal GitHub profile.
    2. Next, navigate to the fork in the browser - and click the "Code" button.
    3. Then, select "Codespaces" and click the "+" button to create a new codespace.
    4. You should see a Visual Studio Code IDE in a new tab - wait till it loads.
    5. You should see a Visual Studio Code terminal open - wait for active prompt.

    Congratulations! Your development environment is ready to use! ‚úÖ

    1.3 Launch Instructions¶

    You need an easy way to navigate instructions, search for relevant information, and copy-paste required code or instructions. This repository is instrumented with an mkdocs.yml that renders the docs in an interactive website format with a single command:

    1. Start by opening the VS Code terminal in your active Codespaces tab
    2. Run this command - and watch for a pop-up dialog to appear (in a few seconds). 
      nohup mkdocs serve > /dev/null 2>&1 &
    3. Confirm "Open in Browser" in dialog - you should see a website like this:
    4. Click Launch Instructions in Table Of Contents to get to this same step.
    5. Now continue following instructions using the web-based workshop guide

    Congratulations! Your workshop guide is ready for you to explore! ‚úÖ

    2. Workshop Outline¶

    Everyone learns differently. Here are three things that can help you apply different learning techniques to build your intuition for the Azure Developer CLI tools and workflows.

    2.1 Application Scenario¶

    Let's start by defining an application scenario for the solution we want. Having a concrete use case allows us to contextualize what we learn in a more practical way.

    SCENARIO: RETAIL CUSTOMER SUPPORT CHATBOT

    You are an AI engineer working for an enterprise retail company that sells products on their online website. You have been asked to build an AI chatbot that has a web-based UI/UX and can answer questions grounded in your product data. Your team wants to make this an AI Agent application - and wants to make sure the architecture is setup to allow end-to-end observability for trustworthy AI delivery. What do you do?

    2.2 Home-Building Analogy¶

    Where do you even start? One way to tackle a complex problem is to find a relevant analogy that can help you reframe issues or solutions in more familiar ways. Let's try this out.

    ANALOGY: BUILDING AI SOLUTIONS IS LIKE BUILDING A HOME

    1. I need a general contractor - Instead of doing everything myself, I need an expert who can manage the resources and schedules, and execute the plan. This is the Azure Developer CLI (azd).
    2. I need a design blueprint - Instead of starting from scratch, I want to look for a "starter" home design that I like and give that to my contractor as the basis to build on. This is your azd template.
    3. I need building materials - I need to decide what fixture and utilities I need and provide my preferences on how they should be installed and configured. This is your infrastrcture-as-code (infra/ folder).
    4. I need interior design - Once the house is in place, I need to decorate it with my things to provide the right user experience for my visitors. This is your application code (src/ folder).
    5. I need a build schedule - This is the construction plan and manifest showing me how the blueprint maps to the materials, and outlining the order in which things get done. This is your configuration file (azure.yaml)
    6. I need to store permits & status - This is how I track the current state of my construction and keep information about various accounts for status checks. _This is your .azure folder (with a .env file)

    You can now make other connections. For instance:

    1. Starter templates are like starter homes. Use them to get familiar with maintenance.
    2. Template customization is like updating your current home to meet new requirements.
    3. Template discovery is like moving to a new home when your current one gets outdated.
